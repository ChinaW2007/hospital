## 项目概述

### 系统功能
医院药品运输系统是一个智能化的药品配送解决方案，包含：
- HIS系统（医院信息系统）：处方管理、药品信息
- hospital_back（后端服务）：ROS通信、药品发送逻辑、音频播报
- hospital_front（前端界面）：处方展示、运输流程可视化
- ROS机器人系统：自动导航、机械臂抓药、送药

### 系统架构
```
┌─────────────┐         ┌──────────────┐         ┌─────────────┐
│  HIS系统     │ ←────→ │ hospital_back│ ←────→ │  ROS机器人  │
│ (MySQL DB)  │         │  (FastAPI)   │         │  (Python)   │
└─────────────┘         └──────────────┘         └─────────────┘
       ↓                        ↓                       ↓
┌─────────────┐         ┌──────────────┐         ┌─────────────┐
│ HIS前端     │         │ hospital_front│         │  机械臂     │
│ (Vue.js)    │         │   (Vue.js)   │         │  (MyCobot)  │
└─────────────┘         └──────────────┘         └─────────────┘
```

### 技术栈
- **后端**：Python 3.x, FastAPI, WebSocket, asyncio
- **数据库**：MySQL (HIS), SQLite (本地状态)
- **前端**：Vue.js, Element UI
- **ROS**：ROS1 Noetic, move_base, pymycobot
- **硬件**：MyCobot机械臂, 小车底盘

## 系统核心功能

### 1. 药品发送逻辑（HIS Sender）

#### 功能描述
逐个药品发送坐标信息给ROS机器人，支持多药品处方。

#### 关键文件
- `hospital_back/app/services/his_sender.py`：核心发送逻辑
- `hospital_back/app/services/ros_listener.py`：ROS消息监听处理

#### 发送流程（最新修改）
```
药品1:
  HIS Sender: 发送 start → start → running → running...
  ROS端: 完成任务（Step 1-5）→ 发送药单级running-step5-return
         格式: {prescription_code}_running-step5-return
  HIS Sender: 收到 → notify_prescription_step5_return() → 发送 end（两次）
  ROS端: 收到end → 发送药品级end消息
         格式: {medicine_id}_{prescription_code}_end
  HIS Sender: 收到 → notify_medicine_completed() → 切换到药品2

药品2:
  HIS Sender: 发送 start → start → running → running...
  ...
```

#### 关键状态变量
```python
# HIS Sender全局状态（his_sender.py）
_current_prescription_code: str = None  # 当前处方编码
_medicine_list: List[Dict] = []         # 药品列表（含坐标）
_current_medicine_index: int = 0        # 当前药品索引（从0开始）
_medicine_total: int = 0                # 药品总数
_medicine_started: Dict[int, bool] = {} # 药品是否收到running-started

# 新增：end消息发送相关状态
_medicine_need_send_end: bool = False   # 是否需要发送end
_medicine_end_data: Dict = {}           # end消息数据
_task_completed: bool = False           # 任务是否完成
```

#### 消息格式
```python
# HIS Sender发送给ROS的消息（/his_sub topic）
{
    "data": "start" 或 "running" 或 "end",
    "prescription_code": "处方编码",
    "medicine_id": 药品ID,
    "x": X坐标,
    "y": Y坐标,
    "z": Z坐标,
    "yaw": yaw角度,
    "medicine_total": 药品总数,
    "medicine_index": 当前药品序号（从1开始）
}
```

#### 关键函数
- `send_medicine_to_ros()`：发送药品坐标（支持start/running切换）
- `send_medicine_end_to_ros()`：发送药品完成信号（end，两次）
- `switch_to_next_medicine()`：切换到下一个药品
- `notify_prescription_step5_return()`：处理药单级running-step5-return
- `notify_medicine_completed()`：处理药品级end消息，触发切换

### 2. ROS消息监听（ROS Listener）

#### 功能描述
监听ROS返回的状态消息，处理药品运输流程的状态更新。

#### 关键文件
- `hospital_back/app/services/ros_listener.py`

#### ROS消息格式
ROS返回的消息格式为字符串：`{medicine_id}_{prescription_code}_{status}`

例如：
- `1_012026070600125_running-started`（药品级消息）
- `012026070600125_running-step5-return`（药单级消息，不包含medicine_id）
- `1_012026070600125_end`（药品级end消息）

#### 消息处理逻辑（最新修改）
```python
# 药单级消息：{prescription_code}_running-step5-return
if medicine_id is None and prescription_code:
    # 触发：通知HIS Sender发送end消息
    notify_prescription_step5_return(prescription_code)

# 药品级消息：{medicine_id}_{prescription_code}_end
elif status == "end" and medicine_id is not None:
    # 触发：通知HIS Sender切换到下一个药品
    notify_medicine_completed(medicine_id, prescription_code)
```

### 3. 音频播报（Audio Service）

#### 功能描述
在特定运输节点触发语音播报，提醒相关人员。

#### 触发时机
- **audio_id=15 (car_can_go)**：单子开始时触发一次（第一个药品收到running-started）
- **audio_id=14 (car_already_arrive)**：单子完成时触发两次（最后一个药品收到end）

#### 关键文件
- `hospital_back/app/services/audio_service.py`
- `hospital_back/app/services/ros_listener.py`

#### 配置
- 摄像头端口：8080（config.py）
- 播报间隔：2秒

### 4. 数据库管理

#### HIS MySQL数据库
- **数据库地址**：192.168.51.133:3306
- **关键表**：
  - `prescriptions`：处方信息（包含status字段）
  - `prescription_items`：药品详细信息（包含坐标）
  - `medicines`：药品基础信息

#### 本地SQLite数据库
- **文件**：`prescription_workflow_state.db`
- **表结构**：`prescription_workflow_state`
  - `prescription_code`：处方编码
  - `current_node`：当前节点
  - `node2_status`：节点2状态（任务确认）
  - `node3_status`：节点3状态（扫码复核）
  - `node4_status`：节点4状态（站台交互）
  - `ros_status`：ROS状态消息
  - `medicine_id`：药品ID

#### 数据库更新逻辑
- 使用UPSERT语法：`INSERT OR REPLACE`
- ROS状态消息解析后自动更新数据库

## 重要约定和约束（Hard Constraints）

### 1. ROS WebSocket配置
- ROS WebSocket listener必须随hospital_back服务启动自动启动（python app.py）
- 配置集中管理：`app/core/config.py`

### 2. 音频播报规则
- Camera audio broadcast (audio_id=15)必须只在第一个药品收到running-started时触发一次
- Camera audio broadcast (audio_id=14)必须只在最后一个药品收到end时触发两次（间隔2秒）
- 摄像头端口：8080

### 3. 药品运输节点命名
- 开具处方（节点1）：处方存在即为完成
- 任务确认（节点2）：收到all_completed消息
- 扫码复核（节点3）：两次扫码完成
- 站台交互（节点4）：两次扫码完成

### 4. 已发放处方判定标准
四个运输节点全部completed：
- 开具处方：处方存在即为完成
- 任务确认：收到all_completed消息
- 扫码复核：两次扫码完成
- 站台交互：两次扫码完成

统计方法：查询本地SQLite prescription_workflow_state表，统计node2_status='completed' AND node3_status='completed' AND node4_status='completed'的记录数量。

### 5. HIS MySQL处方状态更新时机
HIS MySQL prescriptions表status更新为'dispensed'只在四个节点全部完成时（不仅收到end消息）。

### 6. 药品发送规则
- HIS Sender发送start + prescription_code正好两次，然后running + prescription_code持续发送
- HIS Sender重置传输计数器在prescription_code更新时
- HIS Sender发送药品坐标串行按处方：等待前一个药品的{medicine_id}_{prescription_code}_running-step5-return
- HIS Sender发送消息格式必须包含：data, prescription_code, medicine_id, x, y, z, yaw, medicine_total, medicine_index

### 7. 药品独立传输计数
每个药品在处方中有独立传输计数：前2次发送'start'，后续发送'running'。

### 8. ROS消息格式约定
- ROS消息从hospital_back必须使用格式：{medicine_id}_{prescription_code}_{status}
- 例如：11_012026070600125_running-step5-return
- ROS消息必须支持横线分隔格式：running-step1-navigate-to-pharmacy

### 9. 药品过滤规则
HIS Sender必须过滤掉medicine_id=0或NULL的药品：跳过添加到药品列表和跳过发送。

### 10. 数据真实性要求
所有发送消息中的数据必须是真实数据，不能包含0值（除非确实为0）。

### 11. 状态严格匹配
- HIS Sender只更新started状态为True当收到running-started消息匹配当前prescription_code和medicine_id
- HIS Sender只切换到下一个药品当收到running-step5-return消息匹配当前prescription_code和medicine_id

## 当前问题和解决方案

### 问题1：ROS端和HIS Sender逻辑不匹配

**问题描述**：
- ROS端发送药品级running-step5-return（包含medicine_id）
- HIS Sender期望药单级running-step5-return（不包含medicine_id）
- 导致消息处理逻辑混乱，第二个药品无法正确发送

**解决方案**：
- 修改ROS端脚本，使用新创建的`more_task_node629_topic_direction_2026_07_06_3_modified.py`
- ROS端发送药单级消息：{prescription_code}_running-step5-return
- ROS端处理HIS Sender的end消息，返回药品级end：{medicine_id}_{prescription_code}_end

**验证方法**：
1. ROS端日志显示：`Published prescription-level running-step5-return: {处方编码}_running-step5-return`
2. ROS端收到end后日志显示：`Published medicine-level end message: {药品ID}_{处方编码}_end`
3. HIS Sender成功切换到药品2并发送start

### 问题2：数据库连接失败

**问题描述**：
- HIS MySQL连接失败导致处方查询返回空
- HIS Sender发送无效数据（medicine_id=0, xyz=0.0）

**解决方案**：
- HIS Sender添加MySQL连接重试机制（30秒间隔）
- 检查查询结果，数据为空时停止发送并重置状态
- 添加详细日志记录查询过程和结果

**代码位置**：
- `hospital_back/app/services/his_sender.py`（第505-570行）

### 问题3：全局变量声明不完整

**问题描述**：
- HIS Sender主循环中使用全局变量但未声明global关键字
- 导致"referenced before assignment"错误

**解决方案**：
- 在主循环开始添加完整global声明：
```python
global _current_prescription_code, _medicine_list, _medicine_total, _current_medicine_index
global _medicine_started, _task_completed
global _medicine_need_send_end, _medicine_end_data
```

**代码位置**：
- `hospital_back/app/services/his_sender.py`（第580-590行）

### 问题4：启动脚本Python命令错误

**问题描述**：
- restart-all.ps1使用`python app.py`命令
- Windows系统需要使用`py app.py`命令

**解决方案**：
- 修改restart-all.ps1脚本，使用`py app.py`
- 位置：第73行和第198行

**文件位置**：
- `d:\Demos\hospital\restart-all.ps1`

### 问题5：SQL语法错误

**问题描述**：
- HIS Sender使用'?占位符在pymysql查询中
- pymysql使用'%s'占位符，导致查询失败

**解决方案**：
- 修改所有SQL查询，使用'%s'占位符
- 添加详细日志记录查询语句和参数

**代码位置**：
- `hospital_back/app/services/his_sender.py`（第505-570行）

## 教训和经验（Lessons Learned）

### 1. 后端集成优于独立服务
ROS WebSocket和音频播报功能集成到hospital_back服务，确保统一管理，避免独立服务的配置复杂性。

### 2. 连接失败必须有重试机制
MySQL和WebSocket连接失败必须包含定期重试（30秒间隔），避免永久缓存错误导致服务停止。

### 3. 消息类型必须匹配
ROS消息类型必须使用正确的消息定义（如'his_sub/HisSub'而非'std_msgs/String'），避免字段验证错误。

### 4. 数据库查询使用ORM或明确列名
使用列索引（如ws_row[2])容易出错，应使用ORM或明确列名查询，避免索引错位。

### 5. 全局变量声明必须完整
在函数中使用全局变量时，必须声明所有需要的变量，避免"referenced before assignment"错误。

### 6. SQL语法必须正确
pymysql使用'%s'占位符，不是'?，查询语法错误会导致结果为空。

### 7. GROUP BY防止重复数据
药品位置查询必须使用GROUP BY pi.medicine_id，避免重复条目导致medicine_total计数错误。

### 8. 时序同步至关重要
多药品发送时，必须确保ROS端和HIS Sender的时序同步，避免消息处理冲突导致系统卡住。

```python
def notify_prescription_step5_return(prescription_code: str):
    """
    处理药单级running-step5-return，设置end发送标志
    """
    global _current_prescription_code, _current_medicine_index, _medicine_list
    global _medicine_need_send_end, _medicine_end_data, _medicine_total
    
    if prescription_code != _current_prescription_code:
        print(f"[HIS Sender] [ERROR] 处方编码不匹配！")
        return
    
    if _current_medicine_index < len(_medicine_list):
        current_medicine = _medicine_list[_current_medicine_index]
        current_medicine_id = current_medicine["medicine_id"]
        
        _medicine_need_send_end = True
        _medicine_end_data = {
            "prescription_code": prescription_code,
            "medicine_id": current_medicine_id,
            "medicine_data": current_medicine,
            "medicine_index": _current_medicine_index + 1,
            "medicine_total": _medicine_total
        }
        
        print(f"[HIS Sender] 已设置 end 发送标志")
```
